# Versioning

Project365 follows **Semantic Versioning** (SemVer): `MAJOR.MINOR.PATCH`

- **MAJOR** — incompatible changes or complete releases
- **MINOR** — new features, backward-compatible
- **PATCH** — bug fixes, backward-compatible

## Release Strategy

### Release A — Foundation

The current release. Establishes the project structure, documentation, standards, and workflow. All files are created and organized.

### Release B — Lesson Packages

Automated lesson generation, enhanced templates, structured Lesson Packs for all 365 days, resource library expansion.

### Release C — CLI

Command-line interface with commands for daily workflow: `start-day`, `end-day`, `generate-lesson`, `review`, `status`.

### Release D — Dashboard

Web-based dashboard for progress tracking, statistics, visualizations, and learning analytics.

## Current Version

**v2.0.0** — Foundation release with complete documentation and standards.
